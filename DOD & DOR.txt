Definition of Done:
- code compiles without errors
- the feature is developed and tested
- the feature is ready to be sent to the customer
- the product owner accepted the feature
- working in all the platforms 
- the feature doesn't affect other features 

Definition of Ready:
- the user story must be clear as all the team members understand what it means
- the user story must be testable if there is an effective way to determine if the functionality works as expected
- the user story must be feasible if it can be completed in one sprint
